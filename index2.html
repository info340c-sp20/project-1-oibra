I've been a TA for CSE 14X for the past two years (I am not currently), and in that time I've seen a lot of the student pain points for that class. I'm also on r/udub, so I've seen what students complain the most about there. The most common complaint/warning about the 14X series is the harsh internal grading on style. A lot of the course staff feel that we are pretty clear and give a lot of resources to students on writing "well-styled" (by our standards) code, but students' sentiments towards the courses would suggest otherwise, that they still feel that grading code internally on style is not entirely transparent or fair.

This web app will function as an educational Java linter that checks the style of users' input Java code and inform users of what the style errors are with suggestions on how to fix them. This app will be built around the style guidelines for UW CSE 14X (Intro CS1 & 2 at UW) with those students as the primary audience, but should be a helpful resource for any novice Java learners. The project proposal can be viewed here.
