Java Style Linter

This web app functions as an educational Java linter that checks the style of usersâ€™ input Java code and inform users of what stylistic issues the code has with suggestions on how to fix them. This app is built around the style guidelines for UW CSE 14X (Intro CS1 & 2 at UW) with those students as the primary audience, but should be a helpful resource for any novice Java learners. Be aware though, that if you are not a student in those classes, make sure that your code fits the quality guidelines of your organization and that they're consistent with the guidelines used by this tool if you want to use it.

This tool is meant to provide a lot of information, but makes no guarantees that what is found by this tool will encompass all possible style issues in the input code. In fact, while this tool is made to catch a lot of style issues, creating a tool to catch every possible style error is nearly impossible and certainly not something I tried to do, so make sure to give your code a second look yourself.

Indentation

Line 4
int Var = 6;

This line is indented incorrectly.

You should indent your program one tab further every time you open a curly brace and indent one tab less every time you close a curly brace. This line should be indented more.

Long Lines

Line 3
   // this is a really long comment that should be picked up as being too long by the style checker because it's so long

This line is too long (over 100 characters).

Lines of code should ideally max out at 80 characters, and should never exceed 100 characters in length. Lines that are too long should be broken up and wrapped to the next line.

Naming Conventions

Line 1
public static final int constant = 1;

This constant name has the wrong naming conventions.

Constant names should be SCREAMING_CASED, in all uppercase letters with words separated by underscores.

Printing

Line 5
System.out.print("hello world\n");

\n should only be used in printf.

\n should only be used in printf statements, and only once at the end of the line. Otherwise, you should just use println if you want to print a full line.

Line 5
System.out.print("hello world\n"); System.out.println("");

These printing statements can probably be combined.

Rather than have a bunch of adjacent print statements, you should combine them into one where you can. Each full line can be combined into one println statement.

Line 6
System.out.println("");

A blank println should actually be blank.

You should always print a blank line using System.out.println(). Printing an empty String (System.out.println("")) is considered bad style; it makes the intention less clear.
